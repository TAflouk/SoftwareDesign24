About HTML Tags

All pages are made up of tags. Every object on a webpage – a heading, an image, a link, a list – all of these objects have their own tags.

Look at any webpage address, you will see (some browsers now hide it though) on the left hand side either http: or https. The http part is:

Hyper Text Transfer Protocol. The "s" is added for Secure meaning that the site has some security in the form of encryption.

Hyper Text:

So all pages are created from hypertext.  They get sent across the Internet using a transfer protocol. 

When you put in a url (uniform resource locator) or web address, a request goes to a server where that site lives, and the server returns the requested page if it is found.  Your computer downloads the page and it's contents, and the browser renders it.

HyperText is written in a markup language. A markup language is one that uses angle brackets to communicate meaning. A markup language can be used for anything, even a person. For example in XML:

<person>

<head>

<eyes> 2 x brown </eyes>
<mouths>Yes </mouths>

</head>

<body>
A box of chocolates
</body>

</person>

HTML

This idea of structuring is applied is present in HTML where the W3C define what "things" we can define. There are dozens or hundreds of tags, but we'll just use a selection of the main and most useful ones.

All of the main tags we will use are in the Tag Summary page.

An Example of a heading and a paragraph are shown here:

<h2> Some Heading </h2>

<p>This is a paragraph of text</p >

<p>Another paragraph of text</p >

Note that each opening tag has a corresponding closing tag which has a "/" after the opening angle brace ( this thing '<' )

What goes in between is the content. It's often text but there can be other objects that are placed between tags as well.

Nesting of HTML

The easiest way to understand nesting is to think of HTML tags as boxes that hold your content. Your content can include text, images, and related media. HTML tags are the boxes around the content. Sometimes, you need to places boxes inside of other boxes. Those "inner" boxes are nested inside of others.

If you look at the sample page, you will notice that there are lots of elements nested inside other elements. In other words, more tags are opened and closed between the opening <html> and closing </html>. This is called nesting.

You'll see more about the significance of this in the page about the DOM.

If you have a block of text that you want bold inside a paragraph, you'll have two HTML elements as well as the text itself.

<p>Example: This is a <strong>sentence </strong> of text.</p>

As you can see, we have one box (the paragraph) which contains the content of the sentence, plus a second box (the strong tag pair), which renders that word as bold.

When you nest tags, close the tags in the opposite order that you opened them. You open the <p> first, followed by the <strong>, which means you reverse that and close the </strong> and then the </p>.

Another way to think about this is to once again use the analogy of boxes. If you place a box inside another box, you have to close the inner one before you can close the outer or containing box.

What is CSS?

Definition:

  • Cascading Style Sheets is a style sheet language used for describing the presentation of a document written in a markup language such as HTML.
  • CSS is a cornerstone technology of the World Wide Web, alongside HTML and JavaScript.
  • CSS is designed to enable the separation of presentation and content, including layout, colors, and fonts.
  • CSS describes how HTML elements are to be displayed on screen, paper, or in other media
  • CSS saves a lot of work. It can control the layout of multiple web pages all at once
  • External stylesheets are stored in CSS files

What's all this then?

  • HTML provides a way to place many different types of objects onto the web page.
  • CSS is a styling language
  • Bootstrap is a framework with templates creating "things" (like panels, menus etc.) with html and CSS that can be used in Responsive Web Design (RWD). RWD means the site adapts (responds) to display optimally on many devices.
  • JavaScript (JS) is the scripting language for the web. It is an interpreted language and add functionality to a website, allowing elements to be changed "in-place" or sending forms, hiding or displaying new elements. You could even have a game rendered in the webpage written entirely in JS.

We only have to do the first two of these, but depending on time we will often do some Bootstrap and/or JavaScript.

CSS Syntax:

The syntax for CSS is similar to the syntax for languages such as Java or C.

The selector may be:

  • A html tag like h1, h2, p, body etc. All the objects we have seen in the tags page.
  • An id (more info below)
  • A class (more info below)

Note that:

  • {Curly braces} denote the start and end of the block
  • The colon : denotes a property being assigned a value
  • The semi-colon denotes "end";

Important points to note

If you want to style many elements with the same properties and values, you can do it like this: h1, h2, h3, #special, .bluey {color: blue; font-family: Arial}

By using commas to separate the selectors, we can apply the attributes to ALL of the selectors at the same time.

In the example above:

  • Hashtag # denotes an id
  • dot . denotes a class

Targeting

What if we have one unordered list in the nav and another in the main body of the page, but we want to style them differently?

We have two (or more) options. We can:

Specify where the element lives if the one we want to target can be separated from the others i e.

nav > ul > li {background-color: green; color: white;}

This will target ONLY the unordered li elements that live in the nav.

Using Class and ID

An ID is:

  • A unique identifier that appears only ONCE on a page
  • there can be ONLY ONE

  • In CSS, an ID is denoted by a hashtag #

  • <p id = "special"> This is a special paragraph</p> would be styled using the following rule- note the hashtag means id
  • #special {margin-left: 15px; font-weight: italic;}

While the id (identifier) is a way to target that ONE thing on a page, you will often want to target many things- not only on a page but also on an entire site.

A class is a way to package up one or more attributes and we can apply these to many objects, regardless of the type of object.

Say you wanted to style all the images to show as being the same size and with a rounded edge- then you make a class.

<img src = "images\image1.jpg" class="pikrounded">

<img src = "images\image2.png" class="pikrounded"">

<img src = "images\image3.gif" class="pikrounded">

So we have three different images, all of different file formats and possibly of different sizes. We can make them all appear to be the same size

.pikrounded {width: 600px; height: 380px; border-radius: 15px; }
